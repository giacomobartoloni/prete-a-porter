====================
State Management
====================

ChatState
=========

.. autoclass:: chat_orchestrator.state.ChatState
   :members:
   :undoc-members:
   :show-inheritance:

The ChatState extends LangGraph's MessagesState and adds:

- ``session_id``: Unique identifier for the conversation session
- ``next_action``: Control flow indicator ("continue" or "end")

Usage Example
=============

.. code-block:: python

    from chat_orchestrator.state import ChatState
    from langchain_core.messages import HumanMessage
    
    # Initialize state
    initial_state = {
        "messages": [HumanMessage(content="What time is Mass?")],
        "session_id": "sess-abc123",
        "next_action": "continue"
    }
    
    # State after agent processes message
    result = await graph.ainvoke(initial_state, config={
        "configurable": {"thread_id": session_id}
    })

State Fields
============

**messages** (List[BaseMessage])
  Conversation history from LangGraph's MessagesState.
  
  - ``HumanMessage``: User input
  - ``AIMessage``: Assistant response
  - ``ToolMessage``: Tool execution result
  
  Messages are accumulated throughout the conversation.

**session_id** (str)
  Unique identifier for the conversation session.
  
  - Used as thread_id for SQLite checkpointing
  - Included in all logs via context variables
  - Returned in WebSocket responses
  
  Format: ``sess-{uuid}`` (generated by frontend)

**next_action** (Literal["continue", "end"])
  Control flow indicator for conditional edges.
  
  - ``"continue"``: Agent decided to use tools, route to tools_node
  - ``"end"``: No tools needed, end conversation

State Transitions
=================

.. code-block:: text

    Initial State
    (messages=[user_msg])
        |
        v
    agent_node
    (produces AIMessage, sets next_action)
        |
        +--continue--> tools_node
        |              (produces ToolMessages)
        |              |
        |              v
        |           agent_node (loop)
        |              |
        |              +-- if no more tools -->
        |
        +--end--> END

Session Persistence
===================

State is persisted to SQLite after each graph invocation:

.. code-block:: python

    # State is automatically saved
    result = await graph.ainvoke(state, config={
        "configurable": {"thread_id": session_id}
    })
    
    # Later, state can be retrieved
    # (handled by AsyncSqliteSaver checkpointer)

Checkpoint Structure
~~~~~~~~~~~~~~~~~~~~

SQLite stores the complete state under each thread_id:

- ``thread_id``: session_id
- ``messages``: Full message history
- ``next_action``: Last decision
- ``timestamp``: When state was saved

State Updates in Handlers
==========================

In agent nodes, state is updated by returning a dictionary:

.. code-block:: python

    def agent_node(state: ChatState) -> dict:
        # Process state
        llm = _get_llm()
        response = llm.invoke(state["messages"])
        
        # Return update
        return {
            "messages": [response],
            "next_action": "continue" if has_tools else "end"
        }

The returned dictionary is merged into the state by LangGraph.

Type Hints
==========

All state operations are fully type-hinted:

.. code-block:: python

    from typing import Literal
    from langchain_core.messages import BaseMessage
    
    state: ChatState = {
        "messages": [...],
        "session_id": str,
        "next_action": Literal["continue", "end"]
    }
