Root Cause A: 401 Unauthorized — 11 FAILURES

Aspect Detail
What fails 7 homily contract tests (localhost:8002) + 4 liturgy contract tests (localhost:8001)
Error 401 Unauthorized or KeyError: 'result' (liturgy tests return raw error body)
Mechanism Both agents use HTTP Basic Auth via _basic_auth_middleware (packages/a2a-protocol/src/a2a_protocol/server.py:401). Credentials from .env: A2A_BASIC_AUTH_USERNAME=a2a, A2A_BASIC_AUTH_PASSWORD=change-me-in-production. The /health endpoint bypasses auth, so is_agent_available() returns True — but all A2A POST / requests fail.
Why tests don't send auth The make_message_send helper in both test_homily_contract.py:63 and test_liturgy_contract.py:83 doesn't include Authorization headers. The http_client fixture creates a plain httpx.Client(timeout=30.0) with no auth configured.
Fix Add auth=httpx.BasicAuth("a2a", "change-me-in-production") to the http_client fixture and to make_message_send. Either hardcode (from env vars) or read from .env.

Root Cause B: Docker services not healthy — 26 ERRORS

Aspect Detail
What fails All tests depending on the docker_compose session fixture: e2e tests for chat-orchestrator, homily-agent, liturgy-agent, and full flow scenarios
Error Failed: Services did not become healthy within {timeout}s (bug: {timeout} is literal because the string isn't an f-string — contracts/tests/conftest.py:55)
Mechanism The fixture runs docker compose up -d --build (rebuilds images) then polls docker compose ps --format json every 3s for up to 120s. Services filtered: excludes a2a-inspector. All others must have Health: "healthy". The frontend has no health check so it's excluded.
Why it times out --build rebuilds the chat-orchestrator image (UV sync + build: ~30-60s). After rebuild, health checks start after start_period: 10s (chat-orchestrator), with interval: 30s and retries: 3 — up to 100s to become healthy. Combined build + health check exceeds 120s.
Fix Options: (a) increase timeout to 240s, (b) run with --no-docker flag when services are already running, (c) add proper f-string to error message, (d) skip the --build flag for non-image changes.
